# 📐 DecodeLabs UI/UX Internship - Project 2: The Blueprint (Wireframing)

This repository contains the Low-Fidelity Wireframe design for an E-commerce Landing Page, optimized to fix user confusion, reduce clutter, and streamline the conversion funnel.

---

## 📌 Scenario & Challenge
An E-commerce brand was losing customers due to a cluttered layout and confusing user flow during the checkout process. The mission was to design a clean, logical homepage "skeleton" focusing strictly on visual hierarchy, zero distractions, and clear Calls-to-Action (CTAs).

## 🎯 Design Rules Followed
* **Color Palette:** Strictly Black, White, and Grey (No brand colors or real images allowed yet).
* **Placeholders:** Visual elements and images are represented using traditional wireframe placeholder boxes with an image icon.
* **Focus:** Eliminating unnecessary clutter and establishing a direct, clear path from landing to shopping.

---

## 🧱 Wireframe Structural Hierarchy & Layout

1. **Clean Navigation Header:** A minimal top bar featuring a hamburger menu, centralized search bar, and a prominent user/cart profile icon to ensure easy navigation access.
2. **High-Conversion Value Proposition:** The main header explicitly states the core brand solution: **"Premium Products. Zero Hassle Checkout."**
3. **Structured E-Commerce Sections:**
   * **Our Best Sellers:** Placed prominently on the left with a highly visible **[ SHOP NOW ]** CTA button to instantly drive user action.
   * **New Arrivals:** Positioned on the right with a bold, dark contrast **[ ADD TO CART ]** button featuring a cursor indicator, reducing friction for quick purchases.
4. **Clutter-Free Information Blocks:** Clear, structured text blocks at the bottom highlighting express features: *"Explore our most wanted items with 3-step fast checkout."*
